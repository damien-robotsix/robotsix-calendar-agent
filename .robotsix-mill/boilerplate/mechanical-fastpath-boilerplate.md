## Pattern: `mechanical draft fast-path` — compact triage SKIP variant for deterministic periodic-agent sources

### When to use

Use the **mechanical draft fast-path** (compact form) instead of the full `triage SKIP` template when:

- The draft comes from a **deterministic periodic agent** (`agent_check`, `audit`, `bc_check`, `completeness_check`, `module_curator`, `security_posture`, `state_sync`, `test_gap`)
- The change is purely mechanical — no design ambiguity, single-scoped, exact file paths given
- The draft is implementation-ready and requires no refinement

This compact form **skips the refine LLM call entirely** (the full `triage SKIP` text is generated by the refine LLM; the fast-path is applied before the LLM is invoked).

### When NOT to use

Use the full `triage SKIP` template (from `triage-skip-boilerplate.md`) instead when:

- The draft comes from a non-deterministic source (`survey`, `health`, `meta`, `robotsix-chat`)
- The draft needs even minimal interpretation before implementation
- You want to include a detailed grounding verification in the triage note

### Template

```
mechanical draft fast-path (deterministic source '[source_name]') — skipped refine LLM | auto-approve: APPROVE — [source_name] (deterministic rule: mill-internal periodic-agent proposal, no design risk)
```

If the source is NOT in the deterministic-source table in `auto-approve-boilerplate.md`, fall back to a change-type-based justification:

```
mechanical draft fast-path — skipped refine LLM | auto-approve: APPROVE — [change-type-based justification from auto-approve-boilerplate.md]
```

### Decision flowchart

1. Is the draft source a deterministic periodic agent? → If no, use full `triage SKIP` template.
2. Is the change purely mechanical with no design ambiguity? → If no, do NOT skip — route to refine.
3. Are all cited file paths confirmed present? → If no, do NOT skip — route to refine.
4. All yes → Use mechanical draft fast-path; skip the refine LLM and route to implement.

### Relationship to `triage-skip-boilerplate.md`

This file is a **companion** to `triage-skip-boilerplate.md`. That file documents the full `triage SKIP` decision with detailed grounding verification; this file documents the compact fast-path variant used when the deterministic source already guarantees the draft is sound. Both achieve the same outcome (skip refine, route to implement), but the fast-path saves an LLM call.

### Deterministic sources eligible for fast-path

Per `auto-approve-boilerplate.md`, the following sources are deterministic periodic agents whose drafts are pre-vetted:

- `agent_check`
- `audit`
- `bc_check`
- `completeness_check`
- `module_curator`
- `security_posture`
- `state_sync`
- `test_gap`

Not eligible (require full triage): `survey`, `health`, `copy_paste`, `trace_review`, `changelog_autofill`, `repo_description_sync`, `triage_boilerplate`.
